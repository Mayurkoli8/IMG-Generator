# BrandPoster AI 🎨

**AI-powered brand-consistent social media poster studio for real estate companies.**

Generate premium Instagram-ready posters, site visit invitations, New Year greetings, property launches, and more — with your brand's identity baked in every pixel.

---

## ✨ Features

- 🏢 **Brand Profiles** — Store name, colors, logo, contact details, social handles
- 📁 **Asset Library** — Upload logos, property photos, reference posters, brochures
- 🪄 **Prompt Studio** — 11 campaign types, 4 aspect ratios, custom fields
- 🤖 **AI Generation** — OpenAI gpt-image-1 with brand-aware prompt enhancement
- 🏷️ **Brand Overlays** — Programmatic logo + contact footer on every image (no relying on AI for text)
- 🔗 **n8n Webhook** — Full REST webhook endpoint for automation
- 📜 **History** — Browse, filter, re-download all past generations
- 🧪 **Webhook Tester** — Live test UI built right into the app

## 🚀 Quick Start (Local)

### 1. Clone & Install

```bash
git clone https://github.com/yourrepo/brandposter-ai.git
cd brandposter-ai
npm install
```

### 2. Configure Environment

```bash
cp .env.example .env
```

Edit `.env` and fill in:

```env
DATABASE_URL="file:./dev.db"
OPENAI_API_KEY="sk-..."
NEXT_PUBLIC_APP_URL="http://localhost:3000"
STORAGE_MODE="local"
```

### 3. Set Up Database

```bash
npx prisma db push
npx ts-node --compiler-options '{"module":"CommonJS"}' prisma/seed.ts
```

### 4. Start Development Server

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000)

## ☁️ Deploy on Render (Production)

### Step 1 — Push to GitHub
```bash
git add . && git commit -m "initial" && git push origin main
```

### Step 2 — Create Render Web Service
1. Go to [render.com](https://render.com) → New → Web Service
2. Connect your GitHub repo
3. Render auto-detects `render.yaml` — confirm settings

### Step 3 — Add Persistent Disk
1. In your Render service → Disks → Add Disk
2. **Mount Path:** `/data`
3. **Size:** 5 GB (upgradable)

### Step 4 — Set Environment Variables
In Render Dashboard → Environment, add:

| Variable | Value |
|---|---|
| `OPENAI_API_KEY` | `sk-...` |
| `NEXT_PUBLIC_APP_URL` | `https://your-app.onrender.com` |
| `DATABASE_URL` | `file:/data/production.db` |
| `STORAGE_MODE` | `local` |
| `WEBHOOK_SECRET` | (auto-generated by render.yaml) |

### Step 5 — Deploy
Click **Deploy** — Render runs `npm install && npm run build` automatically.

The build command runs:
```
prisma generate → prisma db push → next build
```

First deploy seeds the database with all 11 campaign templates.

## 📡 n8n Webhook Integration

### Endpoint
```
POST https://your-app.onrender.com/api/webhook/generate-poster
```

### Request Payload
```json
{
  "requestId": "unique-idempotency-key",
  "brandId": "your-brand-id-from-app",
  "campaignType": "new_year",
  "prompt": "Create a premium Happy New Year greeting",
  "aspectRatio": "1:1",
  "outputFormat": "png",
  "customFields": {
    "price": "₹45 Lakhs onwards",
    "location": "Pune, Wakad",
    "offer": "Free parking for early bookers"
  }
}
```

### Campaign Types
| Key | Label |
|---|---|
| `new_year` | New Year Greeting |
| `festival` | Festival Greeting |
| `property_launch` | Property Launch |
| `offer_promotion` | Offer / Promotion |
| `site_visit` | Site Visit Invitation |
| `possession` | Possession Update |
| `milestone` | Milestone Announcement |
| `brand_awareness` | Brand Awareness |
| `testimonial` | Testimonial Post |
| `project_highlight` | Project Highlight |
| `construction_update` | Construction Progress |

### Aspect Ratios
| Value | Use case |
|---|---|
| `1:1` | Instagram feed square |
| `4:5` | Instagram portrait |
| `9:16` | Stories / Reels |
| `16:9` | Facebook / YouTube |

### Response
```json
{
  "success": true,
  "generationId": "clxxxx",
  "imageUrl": "https://your-app.onrender.com/generated/poster_xxxx.png",
  "thumbnailUrl": "https://your-app.onrender.com/generated/thumb_xxxx.jpg",
  "brandId": "your-brand-id",
  "campaignType": "new_year",
  "promptUsed": "Create a premium real estate New Year greeting...",
  "createdAt": "2025-01-01T00:00:00.000Z",
  "processingTimeMs": 28450
}
```

### n8n HTTP Request Node Setup
1. Method: `POST`
2. URL: `https://your-app.onrender.com/api/webhook/generate-poster`
3. Headers: `Content-Type: application/json`, `x-webhook-secret: your-secret`
4. Body type: `JSON`
5. In next node, use `{{ $json.imageUrl }}` for the poster URL

### Authentication (optional)
Pass the webhook secret as a header:
```
x-webhook-secret: your-secret-from-env
```
Or as Bearer token:
```
Authorization: Bearer your-secret-from-env
```

## 🗄️ Storage Options

### Local (default)
Files saved to `public/uploads/` and `public/generated/`. Works on Render with persistent disk.

### Cloudinary
```env
STORAGE_MODE=cloudinary
CLOUDINARY_CLOUD_NAME=xxx
CLOUDINARY_API_KEY=xxx
CLOUDINARY_API_SECRET=xxx
```

### AWS S3
```env
STORAGE_MODE=s3
AWS_ACCESS_KEY_ID=xxx
AWS_SECRET_ACCESS_KEY=xxx
AWS_REGION=ap-south-1
AWS_S3_BUCKET=brandposter-ai
AWS_S3_PUBLIC_URL=https://your-bucket.s3.amazonaws.com
```

## 🌊 Generation Pipeline

```
User Prompt
    ↓
Brand Profile + Campaign Template (from DB)
    ↓
Enhanced Prompt Builder (prompt-builder.ts)
    ↓
OpenAI gpt-image-1 (openai-image.ts)
    ↓
Raw Generated Image
    ↓
Brand Overlay Engine (image-overlay.ts)
  → Footer bar: Brand Name + Phone + Website + Instagram
  → Logo: Top-left corner with white container
    ↓
Thumbnail Creation (Sharp resize)
    ↓
Storage (local / Cloudinary / S3)
    ↓
DB Record Updated (GenerationJob)
    ↓
Public URL returned to user / n8n
```

## 🔐 Security Notes

- All secrets via environment variables — never hardcoded
- Optional HMAC webhook authentication
- Rate limiting per IP (configurable via `RATE_LIMIT_RPM`)
- File type and size validation on uploads
- Input length limits on prompts
- Idempotency support via `requestId` to prevent duplicate generations
